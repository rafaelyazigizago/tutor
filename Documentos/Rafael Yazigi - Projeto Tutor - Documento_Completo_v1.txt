PARTE 1 — O QUE É O TUTOR AIOS
════════════════════════════════════════════════════════════

O Tutor AIOS é um professor pessoal de inteligência artificial que ingere
fontes de conhecimento (PDFs, futuramente vídeos e repositórios), gera
currículos automaticamente a partir desse conteúdo e conduz aulas
conversacionais em linguagem natural com o aluno.

O sistema foi projetado para ser local-first (roda na máquina do usuário),
privado (dados do aluno não saem para terceiros) e extensível (novos módulos
podem ser adicionados sem quebrar o que já existe).

A visão de longo prazo é um produto SaaS B2B para criadores de cursos online:
o criador faz upload do seu conteúdo, configura uma persona com sua própria
voz e estilo, e seus alunos têm um tutor personalizado que ensina exatamente
do jeito que o criador ensinaria.

────────────────────────────────────────────────────────────
PROPOSTA DE VALOR
────────────────────────────────────────────────────────────

Para o aluno:
"Sem base, você vira dependente de tutorial.
Com base, você cria suas próprias estratégias."

O Tutor não entrega respostas prontas. Ele constrói entendimento progressivo,
detecta lacunas, sugere pré-requisitos e verifica compreensão antes de avançar.

Para o criador de curso (v2.0):
Um tutor com sua voz, seu estilo e seu conhecimento — disponível 24h para
cada aluno, sem você precisar estar presente.

────────────────────────────────────────────────────────────
4.1 ARQUITETURA EVOLUÍDA
────────────────────────────────────────────────────────────

SEPARAÇÃO RACIOCÍNIO / EXECUÇÃO (Inner Loop / Outer Loop)

Princípio: o agente que decide o que ensinar não deve ser o mesmo
que conduz a aula. Se o raciocínio errar, o erro fica contido.

Inner Loop: execução da tarefa atual (Tool Calls, geração de resposta)
Outer Loop: planejamento da próxima tarefa (o que ensinar, qual módulo)

Para o Tutor: CurriculumAgent decide → TutorAgent executa.

MEMÓRIA HÍBRIDA

Padrão atual: só ChromaDB (busca semântica para tudo)
Padrão v2.0:
- MEMORY.md → contexto rápido da sessão atual (sem busca vetorial)
- ChromaDB → busca semântica profunda entre sessões
Benefício: ConversationEngine mais leve para manter histórico da aula.

CAG — CACHE AUGMENTED GENERATION

Problema: alunos fazem perguntas semanticamente iguais em sessões
diferentes, disparando o pipeline completo (latência + custo de API).

Solução: cache semântico Redis antes do ChromaDB.
Fluxo: Query → Redis Cache → (miss) → ChromaDB → LLM → Cache Store
Resultado: respostas em ~50ms para perguntas já feitas antes.
Redis também usado para: rate limiting, cache de sessão.

CONTROLE DE PROFUNDIDADE DE RACIOCÍNIO

Perguntas simples do aluno = raciocínio curto (Groq direto)
Avaliação pedagógica = raciocínio longo (Claude com mais tokens)
Padrão de transição: inserir mensagem antes de cortar raciocínio
em vez de interromper abruptamente.

LLMCLIENT COM FALLBACK — JÁ IMPLEMENTADO

Groq falhou? → tenta Anthropic automaticamente.
Estrutura pronta para adicionar Gemini e OpenAI quando tiver créditos.

────────────────────────────────────────────────────────────
4.2 SEGURANÇA
────────────────────────────────────────────────────────────

PROMPT INJECTION DEFENSE (pendente)
Detectar e bloquear texto do aluno que possa ser interpretado como
instrução ao LLM antes de enviar ao modelo.
Contexto: em conteúdo técnico, alunos podem colar código ou prompts
que confundem o sistema.

INTENT CHECKS DETERMINÍSTICOS (pendente)
Antes de qualquer ferramenta que acesse arquivos, execute código ou
faça chamadas externas: verificar intenção com lógica determinística
(não um segundo LLM). Só executa se intenção for validada.
Origem: arquitetura de segurança do OpenClaws.

Diferença entre os dois:
- Prompt Injection: protege a entrada do usuário
- Intent Checks: protege a execução de ações dos agentes

────────────────────────────────────────────────────────────
4.3 MELHORIAS PEDAGÓGICAS
────────────────────────────────────────────────────────────

CURRICULUM ENGINE — DETECÇÃO DE PRÉ-REQUISITOS
O sistema deve detectar quando o aluno tenta aprender algo avançado
sem ter a base necessária e sugerir trilha de pré-requisitos antes
de avançar.
Exemplo: aluno pede "me ensine Kubernetes" sem saber Docker.
Sistema responde: "Para entender Kubernetes, precisamos primeiro
cobrir Docker. Quer começar por aí?"

LESSON GENERATOR — NOVOS MODOS DE SAÍDA
Além da aula padrão, o LessonGenerator deve suportar:

Modo Mind Map: mapa de ideias e relações entre conceitos.
Útil para revisão rápida antes de uma aula.

Modo Glossário: identifica todos os termos fundamentais do conteúdo
e explica cada um como se o aluno fosse memorizar para uma prova.

Modo Structured by Chapter: divide o PDF em capítulos e transforma
cada um em uma mini aula independente com exemplos práticos e resumo.
Diferença do modo atual: segue a estrutura do documento original,
não só o currículo gerado pela IA.

Modo Exemplo Duplo: para cada conceito, gera dois exemplos —
um conceitual (o que é) e um real-world (onde aparece na prática).
Padrão inspirado no Refactoring Guru.

CONVERSATION ENGINE — VALIDAÇÃO DO ALUNO
O tutor não deve apresentar resposta da IA como verdade absoluta
em conteúdo técnico. Após explicação, verificar com o aluno antes
de avançar: "Isso faz sentido? Tem algo que parece errado?"
Origem: caso do OpenClaw 3D printing — IA errou na orientação
de impressão e engenheiro mecânico precisou corrigir.

RESEARCH AGENT — PROTOCOLO DE PESQUISA PROFUNDA
Quando o aluno pedir para pesquisar um tema:
1. Mapeamento Inicial: fontes recentes, especialistas, referências
2. Análise Profunda: dados reais, tendências, perspectivas diferentes
Inspirado no padrão do Qwen 3.5 Plus "Investigar a Fundo".

────────────────────────────────────────────────────────────
4.5 VOZ E PERSONA (Fase 4)
────────────────────────────────────────────────────────────

PIPELINE DE VOZ
Fluxo: Microfone → STT (Whisper) → ConversationEngine → TTS → Áudio
Latência crítica: usar Groq por sua velocidade (~200ms) para manter
conversa fluida. A cada segundo de atraso, a experiência de voz quebra.
Referência arquitetural: Call → Transcription → LLM → Knowledge → Action.

PERSONA ENGINE
Cada tutor tem uma persona definida por um arquivo .md:
- Estilo de fala (formal, coloquial, técnico)
- Vocabulário preferido
- Abordagem pedagógica
- Exemplos do domínio do criador

Implementação leve: Skills (arquivos Markdown) em vez de PersonaEngine
completo. O LLM lê o arquivo .md e adapta o comportamento.

VOICE CLONE (produto SaaS)
Para a versão B2B: o criador do curso grava amostras de voz,
o sistema treina um clone com ElevenLabs ou XTTS v2.
O aluno ouve o tutor com a voz real do criador do curso.

ChromaDB mantido por privacidade — não migrar para Pinecone.

[DECISÃO v2.0] CAG (Cache Augmented Generation) sobre o pipeline RAG.
Problema: alunos fazem perguntas semanticamente iguais em sessões diferentes,
disparando o pipeline completo desnecessariamente (latência + custo de API).
Solução: cache semântico antes do RAG. Se a pergunta for semanticamente
similar a uma anterior, retorna do cache (~50ms) sem chamar LLM.
Cache persiste entre sessões — útil porque aluno estuda o mesmo conteúdo
em múltiplas sessões.
Implementação: Redis (já planejado para v2.0 SaaS via Upstash) armazena
os pares pergunta-resposta com embeddings para busca semântica.
Fluxo: Query → Semantic Cache → (miss) → ChromaDB → LLM → Cache Store
Origem: vídeo comparando RAG vs CAG.

Filosofia: "memory + execution + extensibility + security, all composable."

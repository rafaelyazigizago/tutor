PARTE 1 — O QUE É O TUTOR AIOS
════════════════════════════════════════════════════════════

O Tutor AIOS é um professor pessoal de inteligência artificial que ingere
fontes de conhecimento (PDFs, futuramente vídeos e repositórios), gera
currículos automaticamente a partir desse conteúdo e conduz aulas
conversacionais em linguagem natural com o aluno.

O sistema foi projetado para ser local-first (roda na máquina do usuário),
privado (dados do aluno não saem para terceiros) e extensível (novos módulos
podem ser adicionados sem quebrar o que já existe).

A visão de longo prazo é um produto SaaS B2B para criadores de cursos online:
o criador faz upload do seu conteúdo, configura uma persona com sua própria
voz e estilo, e seus alunos têm um tutor personalizado que ensina exatamente
do jeito que o criador ensinaria.

────────────────────────────────────────────────────────────
PROPOSTA DE VALOR
────────────────────────────────────────────────────────────

Para o aluno:
"Sem base, você vira dependente de tutorial.
Com base, você cria suas próprias estratégias."

O Tutor não entrega respostas prontas. Ele constrói entendimento progressivo,
detecta lacunas, sugere pré-requisitos e verifica compreensão antes de avançar.

Para o criador de curso (v2.0):
Um tutor com sua voz, seu estilo e seu conhecimento — disponível 24h para
cada aluno, sem você precisar estar presente.

Argumento de soberania de dados (diferencial competitivo v2.0):
Os dados dos alunos não alimentam modelos de terceiros. O conteúdo do curso
fica sob controle do criador, não das Big Techs. Isso é relevante para
criadores que ensinam conteúdo proprietário ou sensível.

PARTE 4 — VISÃO v2.0
════════════════════════════════════════════════════════════

────────────────────────────────────────────────────────────
4.1 ARQUITETURA EVOLUÍDA
────────────────────────────────────────────────────────────

SEPARAÇÃO RACIOCÍNIO / EXECUÇÃO (Inner Loop / Outer Loop)
Agente que decide o que ensinar ≠ agente que conduz a aula.
Inner Loop: execução da tarefa atual
Outer Loop: planejamento da próxima tarefa

MEMÓRIA HÍBRIDA
- MEMORY.md → contexto rápido da sessão atual
- ChromaDB → busca semântica profunda entre sessões

CAG — CACHE AUGMENTED GENERATION
Cache semântico Redis antes do ChromaDB.
Fluxo: Query → Redis Cache → (miss) → ChromaDB → LLM → Cache Store
Resultado: ~50ms para perguntas já feitas antes.

LLMCLIENT COM FALLBACK — JÁ IMPLEMENTADO
Groq falhou? → tenta Anthropic automaticamente.

CONTROLE DE PROFUNDIDADE DE RACIOCÍNIO
Perguntas simples = raciocínio curto (Groq direto)
Avaliação pedagógica = raciocínio longo (Claude)

────────────────────────────────────────────────────────────
4.2 SEGURANÇA
────────────────────────────────────────────────────────────

PROMPT INJECTION DEFENSE (pendente)
Detectar e bloquear texto do aluno que possa ser interpretado como
instrução ao LLM antes de enviar ao modelo.

INTENT CHECKS DETERMINÍSTICOS (pendente)
Antes de qualquer ferramenta que acesse arquivos ou faça chamadas
externas: verificar intenção com lógica determinística.

────────────────────────────────────────────────────────────
4.3 MELHORIAS PEDAGÓGICAS
────────────────────────────────────────────────────────────

CURRICULUM ENGINE — DETECÇÃO DE PRÉ-REQUISITOS
Detectar quando o aluno tenta aprender algo avançado sem base
e sugerir trilha de pré-requisitos antes de avançar.

LESSON GENERATOR — NOVOS MODOS
- Modo Mind Map: mapa de ideias e relações
- Modo Glossário: termos fundamentais para memorização
- Modo Structured by Chapter: mini aulas por capítulo do PDF
- Modo Exemplo Duplo: conceitual + real-world por conceito

CONVERSATION ENGINE — VALIDAÇÃO DO ALUNO
Após explicação técnica, verificar com o aluno antes de avançar.

RESEARCH AGENT — PROTOCOLO DE PESQUISA PROFUNDA
1. Mapeamento Inicial: fontes recentes, especialistas
2. Análise Profunda: dados reais, tendências, perspectivas

MODO ESPECIALISTA SELECIONÁVEL (novo — publicação Jarvis)
O aluno pode escolher o perfil de tutor antes de iniciar a aula.
Ex: "Especialista em Python", "Especialista em Arquitetura de Software"
Cada perfil é um arquivo .md com instruções específicas.
Extensão natural da PersonaEngine.

────────────────────────────────────────────────────────────
4.7 INFRAESTRUTURA SaaS
────────────────────────────────────────────────────────────

Servidor: Oracle Cloud Free Tier (ARM, 24GB RAM, gratuito)
Alternativa futura: Mini PC Intel N100 (~R$800, 16GB RAM, local)

Stack para versão paga:
- Supabase: PostgreSQL + auth + storage multi-usuário
- Clerk: autenticação por aluno
- Stripe: assinaturas recorrentes
- Redis (Upstash): cache semântico CAG + rate limiting
- Cloudflare: DNS + SSL + proteção

Modelo de negócio:
Criador paga assinatura mensal.
Alunos usam sem custo adicional (incluído no plano).

Argumento de venda: soberania de dados.
O conteúdo do criador e os dados dos alunos ficam sob controle
do criador — não alimentam modelos de terceiros.

────────────────────────────────────────────────────────────
4.8 SEGURANÇA PARA SaaS (multi-tenant)
────────────────────────────────────────────────────────────

- Isolamento de dados por usuário
- Proteção de prompts e persona no servidor
- Pentest: prompt injection, path traversal, IDOR
- Rate limiting por usuário e por curso
- Autenticação robusta via Clerk

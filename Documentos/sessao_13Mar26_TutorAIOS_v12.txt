INFRAESTRUTURA v2.0 SaaS — REFERÊNCIAS
════════════════════════════════════════════════════════════

Stack de referência (vídeo "$20/mês startup"):
Claude + Supabase + Vercel + Stripe + Clerk + Pinecone + Redis + Cloudflare
+ GitHub + PostHog + Sentry + Resend

O que aproveitar para o Tutor SaaS v2.0:

[REGISTRO] Supabase — PostgreSQL + auth + storage numa plataforma.
Candidato para substituir SQLite na v2.0 multi-usuário.
Resolveria: auth de alunos, armazenamento de progresso, gestão de sessões.

[REGISTRO] Clerk — autenticação pronta (OAuth, magic link, sessões).
Na v2.0 com múltiplos alunos, evita construir auth do zero.

[REGISTRO] Stripe — pagamentos e assinaturas. Padrão para SaaS.
Relevante quando o produto virar pago.

[REGISTRO] Redis (Upstash) — cache de sessão e rate limiting.
Relevante para v2.0 com múltiplos usuários simultâneos.

O que já temos decidido diferente:
- Vector DB: ChromaDB (não Pinecone) — privacidade dos dados
- Servidor: Oracle Cloud Free Tier (não Vercel) — controle e custo
- Deploy: próprio servidor ARM (não serverless)

Nota: custos de planos gratuitos crescem com usuários.
Planejar upgrade de planos antes do lançamento SaaS.

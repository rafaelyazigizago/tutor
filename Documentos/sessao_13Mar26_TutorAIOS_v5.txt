Atualmente não temos essa proteção.

[DECISÃO v2.0] Híbrido de memória: MEMORY.md + ChromaDB.
Padrão do OpenClaws: arquivo Markdown para contexto de sessão rápido
(sem busca vetorial) + ChromaDB para busca semântica profunda.

[DECISÃO v2.0] Separação Inner Loop / Outer Loop no OrchestratorEngine.
Inner Loop: execução da tarefa atual (Tool Calls & Steering).
Outer Loop: planejamento da próxima tarefa (Four-Step Turn, Three-Tier Queue).
Atualmente o OrchestratorEngine faz isso de forma achatada.
Para v2.0 com múltiplos agentes, essa separação melhora o controle de fluxo.

[REGISTRO] OpenClaws usa Tool Sandboxing + Docker Isolation para segurança
das ferramentas dos agentes. Referência para quando implementarmos isolamento
de execução no Tutor (ex: agente que roda código do aluno).

BLOCO DE CONTEXTO PARA O PRÓXIMO ASSISTENTE
════════════════════════════════════════════════════════════

O projeto Tutor AIOS é um professor pessoal de IA que ingere fontes de
conhecimento (PDFs, futuramente vídeos), gera currículos e conduz aulas
conversacionais em linguagem natural. Desenvolvido por Rafael em Python 3.12.

O projeto estava rodando no Windows 11 (C:\AI\Tutor) com Ollama + qwen2.5:7b,
ChromaDB e sentence-transformers. Backup completo salvo no Google Drive
(ProjTutor_Vrs1_11Mar26_1016.zip). Pipeline ponta a ponta validado e funcional.

A partir desta sessão, o ambiente migra para Linux Zorin 18 num Lenovo IdeaPad
com i7-10750H, 8 GB RAM e GTX 1650. Por limitação de RAM, o projeto adota
estratégia multi-LLM com APIs online: Gemini Flash 2.0 (ingestão), Groq/Llama 3
(conversa), GPT-4o mini (currículo), Claude (raciocínio). Nenhuma chave de API
criada ainda. Ambiente Zorin não verificado ainda.

ALERTA DE SEGURANÇA: antes de qualquer código com APIs, verificar que todas
as chaves estão apenas no .env e que o .gitignore está correto. Implementar
Prompt Injection Defense no ConversationEngine (ainda não existe).

Decisões v2.0 registradas: Skills como PersonaEngine, protocolo de pesquisa
profunda no ResearchAgent, Oracle Cloud como servidor, modos Mind Map /
Glossário / Structured by Chapter no LessonGenerator, exemplo duplo
conceitual+real-world, detecção de pré-requisitos no CurriculumEngine,
híbrido MEMORY.md+ChromaDB, separação Inner/Outer Loop no Orchestrator,
investigar BMAD.
